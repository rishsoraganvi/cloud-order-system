Objective
---------
Create API Gateway (nginx) that routes client requests to appropriate microservices and applies rate limiting. No coding required—configuration only.

Purpose: Single entry point for clients; request distribution and rate limiting.
Output: nginx.conf with full routing, rate limiting, and timeout configs.

<tasks>

<task type="auto">
  <name>Task 1: Create nginx.conf with routing and rate limiting</name>
  <files>gateway/nginx.conf</files>
  <action>
    - Create `gateway/nginx.conf` with:
      * Upstream blocks for each service:
        - user-service:8001
        - product-service:8002
        - order-service:8003
      * `limit_req_zone` directive: 10 req/s per client IP, 10m zone size.
      * Server block on port 80:
        - Location /api/users/ → proxy_pass http://user-service/users/
        - Location /api/products/ → proxy_pass http://product-service/products/
        - Location /api/orders/ → proxy_pass http://order-service/orders/
      * Apply rate limiting to all locations.
      * Proxy settings: proxy_set_header for Host, X-Forwarded-For, X-Forwarded-Proto.
      * Timeouts: proxy_connect_timeout 5s, proxy_read_timeout 30s.
    - Config should support docker-compose service names (user-service, product-service, etc. on internal network).
  </action>
  <verify>
    <automated>grep -q "upstream user_service" gateway/nginx.conf && echo "nginx.conf has upstream blocks"</automated>
  </verify>
  <done>nginx.conf present with routing, rate limiting, and proxy headers configured.</done>
</task>

<task type="auto">
  <name>Task 2: Create Dockerfile for nginx gateway</name>
  <files>gateway/Dockerfile</files>
  <action>
    - Create `gateway/Dockerfile`:
      * FROM nginx:alpine (minimal image)
      * COPY gateway/nginx.conf /etc/nginx/nginx.conf
      * EXPOSE 80
      * CMD ["nginx", "-g", "daemon off;"]
  </action>
  <verify>
    <automated>docker build -t gateway:local gateway/ 2>&1 | grep -q "Successfully" && echo "Gateway image builds"</automated>
  </verify>
  <done>Dockerfile present; image builds successfully in docker-compose.</done>
</task>

</tasks>
