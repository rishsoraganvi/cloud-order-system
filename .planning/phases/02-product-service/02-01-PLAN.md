## Trust Boundaries

Client -> API Gateway -> Product Service (untrusted input)
Order Service -> Product Service (internal, but still validate)

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01 | I | POST /products endpoint | mitigate | Validate ProductRequest with `@Valid` annotation; enforce length limits on name/description. |
| T-02 | I | PUT /products/{id}/reserve | mitigate | Validate qty param > 0; reject invalid product ids early; use `@Transactional` for atomicity. |
| T-03 | T | Stock decrementation | mitigate | Use database transaction with row-level lock (`@Lock`) to prevent race conditions. |
| T-04 | R | Database access | accept | No sensitive auth data in product service; low risk of unauthorized access. |
</threat_model>
