<task type="auto">
  <name>Task 1: Create complete CI/CD workflow with lint, test, build, push jobs</name>
  <files>.github/workflows/ci-cd.yml</files>
  <action>
    - Create `.github/workflows/ci-cd.yml` GitHub Actions workflow with:
      
      **Trigger:**
      - on: [push (branches: [main]), pull_request (branches: [main])]
      
      **Jobs:**
      1. **lint-python** (runs-on: ubuntu-latest)
         - Checkout code
         - Set up Python 3.11
         - Install dependencies: pip install flake8
         - Lint all Python services: flake8 services/{user,order,notification}-service/
      
      2. **test-python** (runs-on: ubuntu-latest)
         - Checkout code
         - Set up Python 3.11
         - For each Python service (user, order, notification):
           * pip install -r services/{service}/requirements.txt
           * pytest services/{service}/tests/ -v
      
      3. **test-java** (runs-on: ubuntu-latest)
         - Checkout code
         - Set up Java 17 (temurin)
         - cd services/product-service && mvn test -q
      
      4. **build-and-push** (runs-on: ubuntu-latest, needs: [lint-python, test-python, test-java])
         - Checkout code
         - Configure Oracle Cloud credentials (from GitHub Secrets)
         - Log in to OCIR
        - For each service:
          * docker build -t {service}:${{ github.sha }} services/{service}/
          * docker tag {service}:${{ github.sha }} ${{ secrets.OCIR_REGISTRY }}/{service}:latest
          * docker push ${{ secrets.OCIR_REGISTRY }}/{service}:latest
         - Similar for gateway (nginx)
  </action>
  <verify>
    <automated>test -f .github/workflows/ci-cd.yml && grep -q "pytest\|mvn test" .github/workflows/ci-cd.yml && echo "CI/CD workflow configured"</automated>
  </verify>
  <done>Workflow file present with all jobs: lint, test (Python and Java), docker build/push configured.</done>
</task>

<threat_model>
## Trust Boundaries

GitHub Actions → Oracle Cloud (secrets-based authentication)
GitHub Actions → Services (running tests on code)

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01 | S | GitHub Secrets (Oracle Cloud keys) | mitigate | Use GitHub-managed secrets (encrypted at rest); rotate keys regularly; use IAM role ARN if available. |
| T-02 | T | Docker image layer caching | mitigate | Invalidate cache if base image or dependencies change; keep Dockerfiles minimal. |
| T-03 | R | Failed tests auto-merge | mitigate | Require branch protection rule: "Dismiss stale PR approvals when new commits are pushed". |
</threat_model>
