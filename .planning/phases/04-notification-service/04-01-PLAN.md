<threat_model>
## Trust Boundaries

RabbitMQ -> Notification Service (untrusted messages — validate event schema)
Notification Service -> External notification providers (email, SMS, webhook)

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01 | I | RabbitMQ event deserialization | mitigate | Validate event JSON schema before processing; reject malformed events. |
| T-02 | T | External API calls (SendGrid, Twilio) | mitigate | Use verified SDK libraries; apply request timeouts; log errors. |
| T-03 | I | Webhook notifier | mitigate | Validate webhook URL format; reject localhost or private IPs; use HTTPS only. |
| T-04 | R | Message loss | accept | RabbitMQ persists queue; if service crashes, message will be redelivered on restart. |
</threat_model>
